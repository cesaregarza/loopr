# Case Studies

These pages show what `loopr` looks like when applied to a real problem rather
than just described as a library.

They sit one layer below the main docs:

- the main docs explain the public API and internals
- the validation docs explain how to evaluate the system
- the case studies show a concrete domain, setup, and observed outcome
- the validation harness docs explain how those reports should be structured

Each case study is intentionally structured with:

1. `Context`
2. `Results`

That keeps them closer to evidence reports than to product tutorials.
